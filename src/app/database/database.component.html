Database

Introduction

  • Az adatbázisok strukturált adat tárolására szolgálnak, és két fő típusa van
    1. Relációs adatbázisok (SQL)
    2. Nem relációs adatbázisok (NoSQL)

Relational database

  • A relációs adatbázisok (RDBMS) táblákban tárolják az adatokat, ahol az adatok előre meghatározott szerkezettel (oszlopok, sorok) rendelkeznek, és kapcsolatok vannak az egyes táblák között
    • RDBMS = Relational DataBase Management System

  • Jellemzők
    1. Táblázatos szerkezet: Az adatok táblákban (sorok, és oszlopok formájában) kerülnek tárolásra
    2. Előre meghatározott séma: Az adatok struktúráját (mezők, adattípusok) előre definiálni kell
    3. Kapcsolatok táblák között: Az adatok közötti kapcsolatokat idegen kulcsok (FOREIGN KEY), és kapcsolati műveletek (JOIN) segítségével kezelik
    4. SQL alapú lekérdezések: Az SQL nyelv segítségével hajthatunk végre adatlekérdezéseket, módosításokat és törléseket
    5. ACID-kompatibilitás: A tranzakciók követik az ACID szabályokat
      • Az ACID egy tranzakciós modell, amely garantálja az adatintegritást, és konzisztenciát a relációs adatbázisokban

        • Leírás
        Atomicity (Atomitás) A tranzakciók vagy teljesen végrehajtódnak, vagy egyáltalán nem (ROLLBACK lehetőség).
        Consistency (Konzisztencia) Az adatbázis mindig érvényes állapotban marad, biztosítva az adatintegritást.
        Isolation (Elszigeteltség) Párhuzamos tranzakciók nem zavarják meg egymást, az adatok inkonzisztens állapota elkerülhető.
        Durability (Tartósság) A végrehajtott tranzakciók nem vesznek el rendszerhiba esetén sem.

    6. Tranzakciókezelés: Több lépésből álló műveletek végrehajtása úgy, hogy vagy minden lépés végrehajtásra kerül, vagy egyik sem (ROLLBACK)
    7. Indexek és normalizálás: A hatékony keresést és adatkonzisztenciát indexek és normalizálási szabályok segítik
    8. Kiváló adatkonzisztencia: Nagy pontosságot és megbízhatóságot biztosít az adatok számára

  • Előnyök
    1. Adatintegritás és konzisztencia
    2. Komplex lekérdezések (JOIN, GROUP BY, ...)
    3. Jó támogatás és széles körű használat

  • Hátrányok
    1. Skálázás nehézségei (vízszintes skálázás korlátai)
    2. Nagy mennyiségű, változó adatra kevésbé hatékony

  • Népszerű relációs adatbázisok
    1. MySQL
      • Nyílt forráskódú, ingyenesen használható (van fizetős verzió is)
      • Nagyon elterjedt a webes fejlesztésekben (WordPress, Joomla, ...)
      • Gyors és egyszerű használat, de nem minden esetben a legjobb választás nagyvállalati alkalmazásokhoz
      • Hátrány: Korlátozott funkcionalitás bonyolult tranzakciókezelés esetén
    2. PostgreSQL
      • Haladó SQL-funkciók, például JSON-kezelés, bővíthetőség
      • Erős ACID és tranzakciókezelés, kiválóan alkalmas banki, és vállalati rendszerekhez
      • Jobb teljesítmény és skálázhatóság, mint MySQL esetén
      • Hátrány: Bonyolultabb beállítás, és konfiguráció
    3. Microsoft SQL Server
      • Microsoft által fejlesztett robusztus adatbáziskezelő
      • Jól integrálható Microsoft termékekkel (például .NET, Azure)
      • Könnyen kezelhető, stabil, skálázható
      • Hátrány: Magas licencköltségek, zárt forráskód
    4. Oracle database
      • Nagyvállalati környezetre optimalizált
      • Skálázható, teljesítményorientált
      • Támogatja a PL/SQL nyelvet, és fejlett tranzakciókezelést
      • Hátrány: Drága, bonyolult konfiguráció

Non-relational database

  • A NoSQL adatbázisok nem használnak fix táblázatokat és strukturált sémát, inkább dokumentumokat, kulcs-érték párokat, gráfokat, vagy oszloporientált modelleket alkalmaznak

  • Jellemzők
    1. Séma nélküli adattárolás: Az adatok változó szerkezetűek lehetnek, nem kell előre meghatározni az adatmezőket
    2. Nagy skálázhatóság: Horizontálisan skálázható, több szerveren könnyen megosztható
    3. BASE elv: Ellentétben az SQL ACID modelljével, inkább az elérhetőséget és teljesítményt helyezi előtérbe
    4. Gyors írás-olvasás műveletek: Nagy adatforgalmat igénylő alkalmazásokhoz kiváló

  • Előnyök
    1. Gyors adatfeldolgozás nagy volumenű adatnál
    2. Rugalmas adatszerkezet
    3. Könnyen skálázható

  • Hátrányok
    1. Gyengébb adatintegritás (nincs erős ACID támogatás)
    2. Kevésbé hatékony komplex lekérdezésekhez

  • Főbb típusok
    1. Dokumentum alapú adatbázisok: JSON szerű tárolás
      • MongoDB
      • CouchDB
      • Firebase Firestore
    2. Kulcs-érték adatbázisok: Gyors keresés
      • Redis
      • DynamoDB
      • Firebase Realtime Database (hierarchikus JSON alapú)
    3. Gráf adatbázisok: Kapcsolati modellek
      • Neo4j
      • ArangoDB
    4. Oszloporientált adatbázisok: Nagy mennyiségű adat analitikája
      • Apache Cassandra
      • HBase

Comparison



    Tulajdonság SQL (Relációs) NoSQL (Nem relációs)
    Adatszerkezet Táblázatos (sorok, oszlopok) Rugalmas (JSON, kulcs-érték, gráf, oszloporientált)
    Kapcsolatok Erős (FOREIGN KEY, JOIN) Általában nincs
    Lekérdezési nyelv SQL (Structured Query Language) Különböző API-k vagy lekérdezési nyelvek
    Adatintegritás ACID tranzakciók BASE elv
    Skálázás Vertikális (nagyobb szerver) Horizontális (több szerver)

    Használati eset SQL (Relációs) NoSQL (Nem relációs)
    Strukturált adatok (például ügyféladatbázis) igen nem
    Komplex kapcsolatok (például több tábla JOIN) igen nem
    Gyors olvasás/írás nagy adatmennyiségnél nem igen
