SEO

Basics

  • Search Engine Optimization
  • A keresőoptimalizálás (SEO) célja, hogy egy weboldal minél jobb helyezést érjen el a keresőmotorok (például a Google) találati listáján
  • Az organikus, tehát a nem fizetett keresési eredményekre koncentrálunk
    • A fizetett hirdetéseket PPC-nek (Pay-Per-Click), vagy SEA-nak (Search Engine Advertising) hívni
      • A fizetett hirdetésekre jóval kevesebben kattintanak rá, a SEO győz körülbelül 30% - 70% arányban
      • A fizetés abbahagyása nagy Lead vesztéssel jár(hat)
        • Lead
          • A Lead egy potenciális ügyfél, aki valamilyen módon érdeklődést mutatott egy termék vagy szolgáltatás iránt
    • A SEO és a PPC együtt alkotják a SEM-et (Search Engine Marketing), azaz a keresőmarketinget

    1. Technikai SEO
    2. Tartalmi SEO
    3. Off-page SEO
  • Számos más dolog is befolyásolja, befolyásolhatja a rangsorolást (például a domain életkora), de a fenti 3 a legfőbb tényező
  • Google Search Console: URL Inspection tab
    • Itt lehet re-crawl-t / re-indexelést kérvényezni

  • Alapfogalmak
    • Crawling
      • Feltérképezés
      • A keresőmotorok, például a Google, webrobotokat (crawler, bot, spider) használnak, amelyek bejárják az internetet, hogy új és frissített oldalakat találjanak
      • Crawler
        • Követi a linkeket egyik oldalról a másikra
        • Letölti az oldal tartalmát
        • Ellenőrzi a technikai részeket (például robots.txt, meta tag-ek)
      • Ha egy oldal blokkolva van a robots.txt fájlban, a Google nem fogja feltérképezni
      • Akkor, ha az oldal gyorsan betöltődik, és megfelelően van felépítve, a crawler hatékonyabban tudja bejárni
      • Abban az esetben a weboldalunkat nem tudják a crawler-ek bejárni, nem fog megtörténni az indexelés, és ha nincs indexelés, nincs rangsorolás sem
      • Detailed SEO Extension
        • Chrome bővítmény, hogy az oldalunk crawl-olható-e, indexelhető-e
      • A Google Search Console-ban (is) megtekinthető, hogy mennyire - jól - képes a Google a site-unkat crawl-olni, indexelni
        • A "Why pages aren't indexed" szekciónál megnézhetjük az esetleges problémákat egy-egy oldal kapcsán
    • Indexing
      • Indexelés
      • Miután a keresőrobot bejárta az oldalt, a következő lépés az indexelés, amely azt jelenti, hogy a keresőmotor feldolgozza és eltárolja az oldal tartalmát az adatbázisában
      • Ha egy oldal "noindex" meta tag-et tartalmaz, akkor a Google nem fogja indexelni, és nem fog megjelenni a keresési eredmények között
        •     <head>
        ...
        <meta name="robots" content="noindex">
        ...
    </head>
    • Ranking
      • Rangsorolás
      • Indexelés során a keresőmotorok feldolgozzák a weboldal tartalmát (szövegek, képek, videók), metaadatait (title, description, structured data) és szerkezetét, majd ezeket adatbázisukba rögzítik, hogy később a keresési eredmények között megjeleníthessék (SERP)
      • A rangsorolásnál a Google több száz tényezőt figyelembe vesz

  • A teljes téma kapcsán fontos, hogy ne akarjuk teljes mélységében, minden apró részletét megismerni a SEO-nak
    • Egy rendkívül összetett és folyamatosan változó terület, amelynek minden apró részletét lehetetlen (és felesleges) teljesen megérteni
    • A főbb elveket szükséges alkalmazni annak érdekében, hogy jobb eredményeket érjünk el a keresőmotorokban
    • Illetve mindig az akutálisan életbe lépő, felkapottabb trendeket kell követni
    • A megvalósítással sokkal több időt töltsünk, mint tanulással

Technical SEO
  • Technikai SEO

  • URL-struktúra és permalinkek
    • XML sitemap és robots.txt fájlok használata (segítenek gyorsabban indexeltetni az oldalt)
      • XML sitemap
        • Fájl, amely a weboldal összes fontos URL-jét tartalmazza, segítve a keresőrobotokat az indexelésben
        • Általában a gyökér könyvtárban helyezkedik el
          •     <?xml version="1.0" encoding="UTF-8"?>
        <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
            <url>
                <loc>https://fagyilap.hu/</loc>
                <lastmod>2024-04-01</lastmod>
                <priority>1.0</priority>
            </url>
            <url>
                <loc>https://fagyilap.hu/fagylaltok</loc>
                <lastmod>2024-04-01</lastmod>
                <priority>0.8</priority>
            </url>
            <url>
                <loc>https://fagyilap.hu/fagylaltok/csokolade</loc>
                <lastmod>2024-04-01</lastmod>
                <priority>0.7</priority>
            </url>
        </urlset>
      • robots.txt
        • Szintén egy fájl, amely megmondja a keresőmotoroknak, hogy mit indexelhetnek
          •     # Általános beállítások
    User-agent: *
    Disallow: /admin/
    Disallow: /cart/
    Allow: /

    # Bingbot kizárása az egész oldalon
    User-agent: Bingbot
    Disallow: /

    # Googlebot csak a 'fagylaltok' kategóriát indexelje
    User-agent: Googlebot
    Disallow: /
    Allow: /fagylaltok/

    # Speciális szabályok a képek indexelésére
    User-agent: Googlebot-Image
    Disallow: /private-images/

    # Sitemap helyének megadása
    Sitemap: https://fagyilap.hu/sitemap.xml
        • A Disallow és Allow direktívák határozzák meg, hogy mely oldalakat indexelhetnek a robotok
        • A Sitemap direktíva biztosítja a keresők számára a sitemap fájl elérhetőségét
    • Rövid, érthető URL-ek, amik tartalmaznak kulcsszavakat
      • Nem https://example-page.com/page=123, hanem https://example-page.com/page=about-us
  • Mobilbarát tartalom
    • A Google Mobile First Indexing-et használ, azaz az oldal mobilverzióját veszi alapul a rangsorolásnál
    • Reszponzív design
    • Megfelelő méretű betűk mobil nézetben
    • Gyors betöltési idő
      • Ellenőrizhető például a PageSpeed Insights-al
  • Oldalsebesség
    • A lassú oldalak rossz felhasználói élményt adnak és hátrébb kerülnek a Google-ben
    • Gyors tárhely használata
    • CSS és JS fájlok minify-olása
    • Cache
      • A tárhelyen (a .htaccess, vagy nginx.conf fájlokban), és az alkalmazásban is be lehet állítani (Service Worker)
        • Service Worker
          • Egy külön JavaScript fájl, amely a böngészőben fut, és lehetővé teszi az offline működést, az erőforrások gyorsítótárba mentését, valamint a háttérben való adatfrissítést
          • Az applikáció elindulásakor regisztrálódik a Service Worker, majd letölt egy előre meghatározott listát az erőforrásokból (például HTML, CSS, JS, képek), és cache-be helyezi őket, a böngésző pedig minden egyes kérésre megpróbálja a cache-ből szolgáltatni az adatokat
      • HTTP fejlécben is beállítható cache 
              fetch('/data', {
        method: 'GET',
        headers: {
            'Cache-Control': 'max-age=3600'  // a böngésző 1 órán keresztül tárolja a választ
        }
    });
        • no-cache: A válasz nem kerül cache-elésre
      • CDN használatával gyorsabbá tehető az oldal betöltése, mivel gyorsítótárazza az oldal tartalmának egy részét
    • A Google szerint ha egy weboldal 1 másodperccel gyorsabban betöltődik, különösen mobil eszközök esetében, akkor a felhasználók nagyobb valószínűséggel (akár 27%-kal valószínűbben) hajtanak végre konverziós műveletet (például vásárolnak, feliratkoznak hírlevélre, kitöltenek egy űrlapot, stb.)
  • Struktúrált adatok
    • Segítenek a Google-nek jobban megérteni az oldal tartalmát, és speciális megjelenéseket biztosítanak (shippet-ek) a keresési találatok között
      • A snippet a Google találati listáján megjelenő rövid leírás
        1. Standard snippet: Alap keresési eredmény
          • A cím, URL és meta leírás jelenik meg
        2. Rich snippet: Speciális információkat tartalmaz
          • A találat mellett egyéb tartalom jelenik, jelenhet meg
          • Például csillagos értékelés, ár, elérhetőség
        3. Featured snippet: A keresési eredmények tetején kiemelt válasz
          • A keresési eredmények tetején megjelenő, a legjobbnak ítélt válasz
    • Több féle formátumot is használhatunk
      1. JSON-LD
      2. Microdata
      3. RDFa

  • Nem szabad elfelejteni, hogy nem akarjuk, hogy minden oldalunk indexelve legyen
    • Nem minden oldal hasznos a keresőmotorok számára, és egyes oldalak indexelése akár negatívan is befolyásolhatja a SEO-t
    • Nem indexelendő oldalak
      1. Login
        • Nem tartalmaz hasznos, nyilvános információt a felhasználók számára
      2. Checkout
        • Ezek dinamikusak és egyéni felhasználói adatokhoz kötődhetnek, nem relevánsak a keresőmotorok számára
      3. Archives
        • Régebbi, már kevésbé releváns tartalmakat tartalmaznak, de továbbra is elérhetők a látogatók számára
        • A kategória archívum oldal is ilyen, tehát az olyan kategória oldal, ami régi, nem releváns
        • Sokszor duplikált, vagy kevésbé hasznos tartalmat tartalmaznak, ami ronthatja az oldal általános SEO teljesítményét

On-page SEO
  • On-page / On-site SEO / Tartalmi SEO

  • Kulcsszókutatás
    • A kulcsszavak azok a kifejezések, amelyeket az emberek a keresőmotorokban használnak
    • A hatékony SEO stratégia alapja a megfelelő kulcsszavak kiválasztása
    • A Google Search Console megmutatja, hogy a weboldalunl milyen kulcsszavakra jelenik meg
    • Több "kutatóeszköz" is létezik
      • Google Keyword Planner
      • Ahrefs
      • SEMrush
      • Ubersuggest
    • Long-tail, és short-tail kulcsszavak egyensúlya
      • Long-tail
        • Hosszú kulcsszó
        • 3 vagy több szóból áll
        • Célzottabb kereséseket fed le
        • Érdemes a long-tail kulcsszavakat megcélozni, mert könnyebb velük rangsorolni, és jobban illeszkednek a keresők szándékához
      • Short-tail
        • Rövid kulcsszó
        • 1-2 szóból áll, nehéz velük jó helyezést elérni
    • Érdemes megértenünk, hogy egy adott kifejezésre keresve a felhasználó hogyan fogja folytatni a keresésé (milyen további szavakkal, szókapcsolatokkal), és az oldalunkon lehetőleg bejárni ezt a teljes utat, azaz minden kérdésére választ adni számára
  • Minőség és relevancia
    • A Google a hasznos, és egyedi tartalmat részesíti előnyben
    • Naprakész, könnyen olvasható, jól struktúrált, tartalmaz kulcsszavakat, képeket, és egyéb multimédiákat
    • Rendszeres frissítés, blog és új tartalom készítése
  • Tartalom
    • Header-ök hierarchikus sorredben következnek (H1, H2, H3, ...)
      • Ajánlott egyetlen H1-et használni egy oldalon
      • A többi címsor korlátlanul használható a tartalom tagolására
      • Semmi sem gátolja, hogy több H1-et használjunk, de SEO szempontból zavarhatja a keresőrobotokat
    • A meta leírás (Meta Description) 150-160 karakter hosszú összegzés az adott page tartalmáról
      • El kell kerülni az idézőjelet, mert a Google-t tördelésre kényszeríti a szöveget
      • Minden page-en egyedi meta-descriptiont kell megadni, ellenkező esetben a crawler "összezavarodhat", lejjebb minősítheti a weboldalt
    • A kiemelések (bold, italic) segítik az olvashatóságot
    • Listák használata
    • Érdemes sok keyword-focused content-et (kulcsszó-orientált tartalom) használni
      • Olyan tartalom, amelyet kifejezetten a keresőoptimalizálás (SEO) érdekében úgy alakítanak ki, hogy a megfelelő kulcsszavakat és kifejezéseket tartalmazza, amelyek a felhasználók által gyakran keresettek az adott témában
      • Akár a teljes website szövegének 80%-a
    • Amiről az adott oldal szól (például fagylalt), meg kell jeleníteni a kulcsszót a linkben, a címben (h1), meta description-ben, a cikk/leírás/oldal első-, és utolsó mondatában
      • Valamilyen variációban érdemes az első h2 tag-be is belecsempészni
        • <h2>Hogyan készül a fagylalt?</h2>
    • Bevethetjük az NLP-t is
      • Natural Language Processing
      • Természetes nyelvfeldolgozás
      • SEO szempontjából az NLP azt jelenti, hogy a keresőmotorok, mint a Google, nemcsak a kulcsszavakat vagy azok pontos előfordulásait nézik a weboldalakon, hanem képesek értelmezni a szövegek valódi jelentését, szándékát és kontextusát is
      • Alkalmazásával a SEO-javítás nemcsak a kulcsszavak alkalmazására összpontosít, hanem a tartalom minőségére és relevanciájára is, hogy jobb és hasznosabb válaszokat nyújtson a felhasználók kereséseire
  • Belső linkelés
    • Segítenek a Google-nek feltérképezni az oldaladat, és erősítik az oldal relevanciáját
    • Érdemes anchor text-et használni
      • Semmi "Kattins ide", vagy Olvasd el a cikket
      • Helyette Tovább a Rólunk oldalra, vagy "Így fejlődtünk az évek során"
      • A Rólunk egy létező oldal, az "Így fejlődtünk az évek során" pedig egy létező cikk, mindkettőnek ez a konkrét neve/címe
    • A weboldal struktúráját úgy kell kialakítani, hogy a felhasználók és a keresőmotorok könnyen hozzáférhessenek a legfontosabb tartalmakhoz, anélkül hogy 3 kattintásnál mélyebbre kellene navigálniuk
      • Nemcsak a keresőoptimalizálás szempontjából fontos, hanem a felhasználói élmény javítása érdekében is
      • Ennél nagyobb mélység esetében a mélyebb szinteket a Google alacsonyabb prioritásként kezeli, és sokkal ritkábban crawl-olja, sőt, megeshet,hogy be sem indexeli őket
  • Képek és multimédia optimalizálása
    • Alt attribútum használata image tag-en
    • WEBP formátumú képek JPG helyett
    • Képek lazy betöltése
    • Az accessibility attribútumok nem közvetlen SEO rangsorolási tényezők, de javítják a felhasználói élményt, és segítik a keresőmotorokat az oldal megértésében

Off-page SEO
  • Off-page / Off-site SEO
    • Az oldalunkon kívül történő tevékenységek, faktorok, amik az oldalunkra vannak hatással
    • Mennyi link (backlink) mutat a weboldalunkra?
      • Minőségi, és releváns weboldalakról érkező hivatkozások
      • Vendégblogolás, PR-cikkek, közösségi média megosztások
      • Facebook, Twitter, LinkedIn, Instagram használata a forgalom növelésére
      • Fórumokon, Reddit-en, és szakmai csoportokban való részvétel
    • A Google úgy tekint a backlink-ekre, mint valamiféle vote-okra
      • Minél több van, annál relevánsabbnak tekinti az oldalunkat
    • A backlink-jeink számának legskálázhatóbb növekedése úgy érhető el, ha értéket teremtünk az oldalunkon, aminek url-jére az emberek rá akarnak kattintani
      • Magas minőségű tartalommal érhető el
    • Általánosan igaz, hogy minél több hiteles, és releváns hivatkozásunk van más oldalaktól, annál jobb helyezést fogunk elérni a Google rangsorában
    • ahrefs.com
      • Megnézhetjük például, hogy a konkurenciának milyen weboldalakra sikerült backlink-eket elhelyeznie

Snippets

  • A Google kereső találati oldalán (SERP = Search Engine Results Page) megjelenő rövid előnézet, amely segít a felhasználónak eldönteni, hogy az adott oldal releváns-e a kereséséhez, miről szól az adott oldal
  • A snippetek célja, hogy növeljék az átkattintási arányt (CTR = Click-Through Rate) azáltal, hogy a felhasználó számára releváns és vonzó információkat mutatnak
    • CTR
      • Mutató, amely azt méri, hogy egy adott keresési találatot vagy hirdetést hányszor kattintanak meg a felhasználók a megjelenések számához képest
      • Minél magasabb a CTR, annál vonzóbb az oldal snippetje a keresési eredményekben
        •             kattintások száma
    CTR = -------------------- × 100%
            megjelenések száma
  • Létrehozásának lépései
    • Kutatómunka, hogy mire szoktak rákeresni az emberek
      • Featured snippetek bizonyos keresésekre jelennek meg csak
    • Átgondolni, hogy hogyan lehet átvenni a jelenlegi helyét (még relevánsabb információkkal megtöltve a tartalmát a snippetnek)
      • A sorban lehet több, a miénknél relevánsabb snippet is
    • Elkészítés
      • Ha mondjuk egy keresésnél megjelenik egy featured snippet tábla formájában, akkor érdemes megnézni, hogy miként rakták ki a weboldalukra azok, akik jelenleg kikerülnek a snippetbe
      • Egyik best practice: H2-nek vagy H3-nak kell szerepelnie az adott weboldalon, és azonnal alatta egyértelmű szöveges választ kell adni 40-60 szóban
        • Definition Snippet lehet belőle
          • Egy tömör, világos válasz, amelyet a Google automatikusan kiemelhet
      • Meg lehet próbálni megmutatni a Google-nek, hogy az oldalunkon van olyan tartalom, ami akár Featured Snippet is lehetne, de ezt a Google dönti el, hogy érdemes-e Featured Snippet-et létrehozni
      • Headerek használata fontos, és alattuk egyértelmű válaszok, leírások
      • A képre érdemes szöveget tenni
  • A kérdés mögé írva kötőjellel a domaint, ahonnan származik a snippet, megnézhetjük, hogy ki a soron következő második találat
    • A Google úgy értelmezi, hogy az adott domainről származó találatokat ki kell hagyni a keresési eredményekből
      •     a fagylalt hőmérséklet -gasztropartner.hu
  • Amennyiben azt szeretnénk, hogy csak egy adott oldal találatai jelenjenek meg, a site: kifejezést használjuk 
          a fagylalt hőmérséklet site:hazipatika.com
  • Mindhárom típusánál megjelenik/megjelenhet a félkövér (bold) szöveg
    • Desktopon, és mobilon is, a 'Math Sans Bold' betűtípus használatával, vagy <b> és <strong> tag is működik, de ezek nem befolyásolják a SEO-t
      • Csak odacsalhatják a szemeket

    1. Standard Snippet
      • Megjelenhet a félkövér szöveg, de a Google automatikusan félkövérítheti a keresett kulcsszavakat a snippetben, még akkor is, ha az eredeti weboldalon nem volt félkövér
    2. Rich Snippet
      • Ha az értékelések, receptek, árak vagy más strukturált adatok között van félkövér szöveg, az esélyes, hogy megjelenik félkövéren
    3. Featured Snippet
      • Jó eséllyel megmarad félkövé, de ha a kereső úgy gondolja, hogy egy adott szó vagy kifejezés fontos, félkövérré teheti a snippetben, függetlenül attól, hogy az oldalon eredetileg félkövér volt-e vagy sem
  • A SERP listában megjelenő tartalmat tudjuk szabályozni
    1. data-nosnippet attribútum
      • Megakadályozza, hogy egy adott HTML elem tartalma megjelenjen a Google találati lista leírásában (Snippet-ben)
      • Csak a megjelölt HTML elemre vonatkozik, nem az egész oldalra
        •     <p data-nosnippet>Ez a szöveg nem jelenik meg a Google snippetben.</p>
    2. nosnippet meta tag
      • Oldalszintű
      • Teljesen megakadályozza, hogy az oldalhoz bármilyen snippet generálódjon a Google találatok között
      • A keresőmotor még indexelheti az oldalt, de nem fog hozzá megjeleníteni leírást/snippetet 
        •     <meta name="googlebot" content="nosnippet">

Standard Snippet
  • A klasszikus keresési találat, amely egy cím, egy URL, és egy leírás (meta description) részből áll



Rich Snippet
  • Strukturált adatokat (JSON-LD, Microdata, RDFa) használva kiegészített találatok, amelyek extra információkat jelenítenek meg
    • Magyarán struktúrált adatot szükséges hozzáadni a weboldalunkhoz
  • Értékelések, események, receptek, ...
  • Rich Result-ként is hivatkoznak rá
  • Magas CTR-t eredményeznek



Featured Snippet
  • Kiemelt találat ("0. pozíció"), amelyet a Google a keresési eredmények tetején jelenít meg
  • Célja, hogy azonnali választ adjon a felhasználók kérdésére, anélkül, hogy rá kellene kattintani a weboldalra
  • Egy jól strukturált oldal tartalmából kerül ki
  • Formátuma lehet
    1. szöveg
    2. lista
    3. táblázat
    4. kép
    5. videó


Structured data

  • Olyan (strukturált / egyértelműen, jól meghatározott módon rendezett) adatokat jelent, amelyek szabványosított formában vannak tárolva, és könnyen feldolgozhatók gépek számára
  • Segít a crawlerek-nek / Google botoknak, hogy úgy jelenjenek meg a keresési találatok, ahogy mi szeretnénk
  • Az oldal tartalmának besorolására / osztályozására való
  • Használatának 3 célja lehet
    1. SEO optimalizálása
      • A strukturált adatok lehetővé teszik, hogy a keresőmotorok pontosabb találatokat jelenítsenek meg
      • Nem helyettesíti a meta tag-eket, és egyéb SEO optimalizálásokat, hanem kiegészíti
    2. Rich Snippet megjelenítése
      • Több információt adnak a felhasználónak, és figyelemfelkeltőbbek lehetnek, mint a hagyományos találatok
    3. Időmegtakarítás
      • Struktúrált adatok használatával gyorsabb, és pontosabb találatokat adhatnak
      • A weboldal tulajdonosának is kevesebb időt kell tölteni az adatok manuális feldolgozásával, vagy optimalizálásával
  • Használatával sem garantált az, hogy a megfelelő meta description fog megjelenni a SERP-en, de egyébként fallback-ként tudja használni a Google, amennyiben nem kapott meta adatot
    • Ha az oldal nem tartalmaz meta description-t, vagy ha a Google úgy ítéli meg, hogy az nem releváns, akkor a Google saját szöveget generálhat a találat leírásaként, amely a keresett kulcsszót tartalmazza
    • a Google ebből a struktúrált adatból próbál információt nyerni, és azt jeleníti meg a meta description helyett a keresési találatokban
  • Google Rich Result Test and Schema Markup Validator

Schema Markup
  • Egy olyan strukturált adat formátum, amelyet a Schema.org nyílt webes szótár használ
  • Lehetővé teszi különböző típusú entitások (például termékek, események, személyek) meghatározását strukturált adatok segítségével
  • A a weboldalon segít megfelelő módon reprezentálni, hogy a keresőmotorok (például Google, Bing) jobban értelmezzék a weboldal tartalmát
    • Ha az adott oldalunkon mondjuk egy recept található, a Schema.org szerint formázva a strukturált adatunkat, a keresőmotorok tudják, hogy ez egy recept, és hogyan jeleníthetik meg mondjuk Rich Snippet-ként
  • Több száz schema markup létezik
    • Avagy a Schema.org szótár több száz különböző Schema Markup típusú adatot tartalmaz

    1. Személyek: név, cím, születési dátum
    2. Termékek: ár, értékelések, elérhetőség
    3. Receptek: hozzávalók, elkészítési idő, receptlépései
    4. Események: dátumok, helyszínek, jegyvásárlás
    5. Cikkek és hírek: cím, szerző, publikálási dátum
    6. Szervezetek: cég neve, tevékenység, elérhetőség
    7. Vélemények, értékelések: csillagos értékelések, vélemények
    8. ...
  • Használhatunk különböző formátumokat (JSON-LD, Microdata, RDFa) a megjelenítésére

  • A primaryImageOfpage a WebPage típusoknál alkalmazandó

JSON-LD
  • JavaScript Object Notation (JSON) for Linked Data
  • Specifikus formátum a Structured Data számára
  • A JSON-LD a Schema Markup egyik legelterjedtebb formátuma
  • JavaScript alapú, és könnyen olvasható
  • Az egyik legújabb, legjobb, és legajánlottabb formátum
  • JSON-t használ, de a Linked Data koncepcióját követi
  • Az adatok a HTML kódon kívül egy (praktikusan egy, nem több) <script> tagben jelenhetnek meg, nem befolyásolják a weboldal vizuális megjelenését
  • Könnyű implementálni
  • A Google és más keresők kifejezetten javasolják a JSON-LD használatát
  • Bonyolultabb adatmodellek esetén is alkalmazható
  • Implementáció után a Google Search Central-ban megtekinthetők a riportok a Structured Data validitását illetően

  • Linked Data
    • A kapcsolt adatok egy olyan módszertan, amely lehetővé teszi az adatok összekapcsolását az interneten keresztül, úgy, hogy a gépek könnyebben megértsék és feldolgozzák azokat
    • 4 alapelve
      1. Egyedi azonosítók: Minden egyes adat (például személy, termék) kapjon egyedi azonosítót (például URL)
      2. Elérhetőség: Az URI-k legyenek elérhetők HTTP-n keresztül, hogy a felhasználók és gépek egyaránt lekérdezhessék azokat
      3. Struktúrált adatok: Az adatok leírására strukturált, gép által olvasható formátumokat kell alkalmazni
      4. Adatkapcsolatok: Az adatok között kapcsolatokat kell teremteni, hogy értelmezhetők legyenek a többi információval együtt

  • Az a legfontosabb, hogy pontos információt adjunk át, ami valóban megfelel az adott oldalnak
    • Ha valami nem áll rendelkezésre, "üresen" ne kerüljön az adathalmazba
  • A túl nagy JSON-LD adatok lassíthatják az oldal betöltési idejét, mivel a weboldal egyes keresőmotorok számára történő indexelését akadályozhatják
    • Az optimális elemszám jellemzően 10.000 - 30.000 közötti érték
    • Semmiképpen se haladja meg a 100.000-res elemszámot

Microdata
  • HTML kiterjesztés, amely a strukturált adatokat közvetlenül a HTML elemekbe ágyazza
  • Könnyen elérhető az oldal tartalmához
  • A weboldal dizájnjával együtt változhat, és a szerkesztése bonyolultabb lehet, mint a JSON-LD esetében
RDFa
  • Resource Description Framework in Attributes
  • RDF szabvány kiterjesztése, amely lehetővé teszi a strukturált adatok beágyazását a HTML dokumentumba
    • RDF
      • Resource Description Framework
      • Egy adatmodell, amelyet a weben található erőforrások leírására, és összekapcsolására használnak
      • A Linked Data egyik alapköve
      • Az RDF adat 3 részből áll (triplet)
        1. Alany (Subject): A szóban forgó doloog (például egy személy, termék)
        2. Állítmány (Predicate): Az adott alanyhoz kapcsolódó tulajdonság, vagy kapcsolat (például a neve)
        3. Tárgy (Object): Az alany tulajdonságának értéke (például Clark Kent)
  • Nehezebben implementálható és karbantartható

Algorithms

  • Az algoritmusok különböző szempontok alapján határozzák meg, hogy mely weboldalak jelennek meg előkelőbb helyen a keresési találatok között
  • A rangsorolásért felelős Google algoritmusok

    • Panda algoritmus (2011)
      • A gyenge minőségű tartalom büntetése
      • A célja a magas színvonalú, jól megírt tartalom előnyben részesítése

    • Penguin algoritmus (2012)
      • A manipulált linkek (link farmok, vásárolt linkek) büntetése
      • A Google figyelembe vette a természetes linképítést

    • Hummingbird algoritmus (2013)
      • A szándék alapú keresés előtérbe helyezése
      • Jobb válaszokat ad a hosszú kérdésekre és a hosszú farok (long-tail) keresésekre

    • Mobilegeddon algoritmus (2015)
      • Mobilbarát webhelyek előnyben részesítése
      • A mobilbarát dizájnok és a gyors betöltési sebesség kulcsfontosságúvá váltak

    • RankBrain algoritmus (2015)
      • A mesterséges intelligencia alkalmazása a keresési eredmények javítására
      • A Google most már képes értelmezni a keresési kifejezések mögötti szándékot

    • BERT algoritmus (2019)
      • A természetes nyelvfeldolgozás (NLP) bevezetése, a keresési kifejezések pontosabb értelmezésére
      • A felhasználói szándékok és a kérdések kontextusa most már sokkal jobban figyelembevételre kerül

    • Core Web Vitals (2021)
      • A felhasználói élmény mérése, különös figyelemmel a betöltési sebességre, interaktivitásra és a vizuális stabilitásra
      • A Largest Contentful Paint (LCP), First Input Delay (FID), és Cumulative Layout Shift (CLS) fontos metrikák

Angular

  • A metaadatok, és a JSON-LD generálása Angular projektekben kulcsfontosságú, különösen SEO szempontjából
  • A dinamikusan betöltődő oldalak esetén az SSR (Server-Side Rendering) használata különösen fontos, hogy a keresőmotorok megfelelően indexeljék a tartalmat, még akkor is, ha az nem statikus HTML
  • A metadatokat, és JSON-LD-t az oldal betöltődésekor, általában az oldal tartalmának lekérése után érdemes generálni
  • A JSON-LD adatokat leginkább a <script type="application/ld+json"> tag-en keresztül adjuk hozzá a <head> szakaszhoz
  • Az Meta szolgáltatás az Angularban egy praktikus eszköz, amely lehetővé teszi, hogy dinamikusan módosítsuk az oldalunk <meta> tag-jeit, amik az oldal metaadatait tartalmazzák
    • Az og: (Open Graph) egy metaadat-kiterjesztés, amelyet a Facebook hozott létre, és amit más közösségi média platformok is átvettek
    • Az Open Graph célja, hogy szabványos formátumban adják meg, hogyan jelenjen meg egy weboldal tartalma, amikor azt megosztják a közösségi médiában

    •     import { Component } from '@angular/core';
    import { Meta, Title } from '@angular/platform-browser';
    import { Router, NavigationStart, NavigationEnd } from '@angular/router';
    import { filter } from 'rxjs/operators';
    import { environment } from '../environments/environment';
    
    @Component({
        selector: 'app-root',
        templateUrl: './app.component.html',
    })
    export class AppComponent {
        siteUrl = environment.siteUrl; // https://fagylaltkiraly.hu
        imgUrl = environment.imgUrl;   // https://img.fagylaltkiraly.hu
    
        constructor(private router: Router, private meta: Meta, private title: Title) {
        this.router.events
            .pipe(filter(event => event instanceof NavigationStart))
            .subscribe(() => {
            this.updateSeoTags();
            });
    
        this.router.events
            .pipe(filter(event => event instanceof NavigationEnd))
            .subscribe((event: NavigationEnd) => {
            this.updateCanonical(event);
            });
        }
    
        private updateSeoTags() { // meta adatok frissítése
        this.title.setTitle('Fagylalt Király - A legfinomabb kézműves fagyik');
    
        this.meta.updateTag({
            name: 'description',
            content:
            'Kóstold meg a Fagylalt Király legjobb kézműves fagylaltjait! Friss, természetes alapanyagokból készült különleges ízek.',
        });
    
        this.meta.updateTag({ property: 'og:title', content: 'Fagylalt Király' });
        this.meta.updateTag({
            property: 'og:description',
            content:
            'Fedezd fel a Fagylalt Király különleges kézműves fagylaltjait, 100% természetes alapanyagokból!',
        });
        this.meta.updateTag({
            property: 'og:url',
            content: this.siteUrl + '/',
        });
        this.meta.updateTag({
            property: 'og:image',
            content: `${this.imgUrl}/assets/images/social_share.jpg`,
        });
        }
    
        private updateCanonical(event: NavigationEnd) { // canonical frissítése
        const canonical = document.querySelector(
            'link[rel="canonical"]'
        ) as HTMLLinkElement;
    
        if (canonical) {
            canonical.href = this.siteUrl + event.url.split('?')[0];
        }

        providedIn: 'root',

        private imgUrl = environment.imgUrl;

        constructor(private meta: Meta, private title: Title) {}

        updateTags(options: {
            title?: string;
            description?: string;
            image?: string;
            url?: string;
        }) {
            if (options.title) {
                this.title.setTitle(options.title);
                this.meta.updateTag({ property: 'og:title', content: options.title });
            }

            if (options.description) {
                this.meta.updateTag({ name: 'description', content: options.description });
                this.meta.updateTag({ property: 'og:description', content: options.description });
            }

            this.meta.updateTag({
                property: 'og:image',
                content: options.image?.startsWith('http')
                            ? options.image
                            : `${this.imgUrl}/${options.image}?w=1200&h=630`,
            });

            this.meta.updateTag({
                property: 'og:url',
                content: options.url?.startsWith('http') ? options.url : `${this.siteUrl}/${options.url}`,
            });
        }

        updateCanonical(event: NavigationEnd) {
            const canonical = document.querySelector('link[rel="canonical"]') as HTMLLinkElement;
            if (canonical) {
            canonical.href = this.siteUrl + event.url.split('?')[0];
            }
        }
    }
